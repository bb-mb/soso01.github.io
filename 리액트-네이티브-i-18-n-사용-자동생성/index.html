리액트 네이티브 i18n 사용, 자동생성

30.09.2021, 1 min read

리액트 네이티브 앱 다국어 지원을 위해서 i18n 라이브러리를 사용해봤다.

i18n이란?

  • Internationalization 의 축약어이다
  • 국제화를 위한 라이브러리
  • 언어 번역 외에도 통화, ltr or rtl 등 지원

RN 적용

i18n에 여러 기능이 있지만, 나는 언어 번역만을 위해서 사용했다.

설정 초기화

1// i18n.js
2import i18n from 'i18n-js';
3import * as RNLocalize from 'react-native-localize';
4

5import { DEFAULT_LOCALE, LANGUAGES, TEXT } from '@constants';
6import * as translations from './locales';
7

8export function setI18n() {
9  i18n.fallbacks = true;
10  i18n.translations = translations;
11  i18n.defaultLocale = DEFAULT_LOCALE;
12  i18n.locale =
13    RNLocalize.findBestAvailableLanguage(LANGUAGES)?.languageTag ||
14    DEFAULT_LOCALE;
15}
16

17// @constants
18export const DEFAULT_LOCALE = 'en';
19export const LANGUAGES = ['en', 'ko', 'ja', 'fr']; // 지원할 국가들
20export const TEXT = {
21  hello: '안녕하세요 {{name}}님',
22  scope1: {
23    scope2: {
24      message: '메세지1',
25      message2: '리액트 네이티브',
26    },
27  },
28};

LANGUAGES 상수에 지원할 언어 목록, TEXT 상수에 앱에서 사용한 문구들을 정의했다.

i18n 설정 초기화는 setI18n 함수에서 정의하고, 스플래쉬 화면을 보여줄 때 setI18n 함수를 호출시켰다. fallback 여부, translations(번역된 언어 문구 객체), 사용자 locale 등을 설정한다.

RN에서 사용자 locale은 react-native-localize 라이브러리를 사용해서 가져왔다. findBestAvailableLanguage 메서드는 인자로 넘겨준 배열의 언어 중에서 사용자에게 가장 적합한 언어를 리턴해준다고 한다. 어떤 알고리즘인지는 잘 모르겠다.

사용

1import { t } from 'i18n-js';
2

3t('scope1.scope2.message2') // 리액트 네이티브

기본적으로 위와 같이 t('key', options) 형태로 사용한다. text.scope1.scope2.message2(options) 와 같이 key를 argument로 넘겨주는 대신 객체에서 접근하는게 더 쓰기 편할 것 같아서 key가 바인딩된 t 함수를 value로 가지는 text 객체를 생성함

1// i18n.js
2export const text = value2Translate(TEXT);
3

4export function value2Translate(translation, scope = []) {
5  return convertObj(translation, convertFunc);
6

7  function convertFunc({ key, value }) {
8    if (typeof value === 'object') {
9      return value2Translate(value, [...scope, key]);
10    }
11    return makeTranslate([...scope, key].join('.'));
12  }
13}
14

15function convertObj(obj, convertFunc) {
16  return Object.keys(obj).reduce((acc, key) => {
17    return { ...acc, [key]: convertFunc({ key, value: obj[key] }) };
18  }, {});
19}
20

21function makeTranslate(key) {
22  return options => i18n.t(key, options);
23}
24

25// Home.js
26import React from 'react';
27

28import { text } from '@i18n';
29import * as S from './Home.styled';
30

31function Home() {
32  return (
33    <S.Wrap>
34      <S.StyledText>{text.hello({ name: 'kim' })}</S.StyledText>
35      <S.StyledText>{text.scope1.scope2.message2()}</S.StyledText>
36    </S.Wrap>
37  );
38}
39

40export default Home;

그리고 {{name}} 은 Interpolation 기능으로 options객체 name의 value로 교체된다.

구글 번역 api로 자동생성

TEXT 상수에서 문구를 한번만 정의하고, 언어별로 번역된 파일은 자동 생성시키기 위해 구글 클라우드 api를 사용했다. 월 요청 50만건까지 무료라서 부담없이 사용할 수 있었다.

1// googleTranslate.js
2const { Translate } = require('@google-cloud/translate').v2;
3const fs = require('fs');
4const path = require('path');
5

6import { LANGUAGES, TEXT } from '@constants';
7import { googleTranslateKey } from './key';
8

9// Instantiates a client
10const translate = new Translate({ key: googleTranslateKey });
11

12(async function run() {
13  saveLocaleJson();
14  saveLocaleIndexJs();
15})();
16

17function saveLocaleIndexJs() {
18  fs.writeFileSync(
19    path.resolve(__dirname, '../src/i18n/locales/index.js'),
20    LANGUAGES.reduce(
21      (acc, lang) =>
22        acc + `export { default as ${lang} } from './${lang}.json';\n`,
23      '',
24    ),
25  );
26}
27

28function saveLocaleJson() {
29  LANGUAGES.forEach(async lang => {
30    const result = await translateObj(TEXT, lang);
31    fs.writeFileSync(
32      path.resolve(__dirname, `../src/i18n/locales/${lang}.json`),
33      JSON.stringify(result),
34    );
35  });
36}
37

38async function translateObj(obj, lang) {
39  const result = {};
40

41  for (const [key, value] of Object.entries(obj)) {
42    result[key] =
43      typeof value === 'object'
44        ? await translateObj(value, lang)
45        : deleteSpanTag(
46            (await translate.translate(addSpanTag(value), lang))[0],
47          );
48  }
49  return result;
50}
51

52function addSpanTag(text) {
53  return text.replaceAll(/{{[^{}]+}}/g, '<span translate="no">$&</span>');
54}
55

56function deleteSpanTag(text) {
57  return text.replaceAll(/<\/?span[^>]*>/g, '');
58}

TEXT 객체의 문구를 언어별로 번역하고, fs 모듈로 직접 json 파일을 생성했다. 리액트 네이티브에서 동적으로 require 경로를 설정하니 에러가 나서 언어별로 생성한 json을 export 하는 index.js도 직접 생성했다. 아마 빌드타임에 require 경로가 정해져야 되는 듯하다.

{{name}} 과 같은 interpolation 구문에서 키값(name)은 번역되지 않도록 번역 전에<span translate="no"> 태그로 감싸고, 번역후에는 span 태그를 삭제시켰다.
